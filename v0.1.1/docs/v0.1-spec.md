# AutoTune FPS v0.1 Spec

## Core Promise

`AutoTune FPS` should feel like:

> "Minecraft optimization for people who do not want to manually tune video settings."

## In Scope

- Fabric client mod targeting `1.21.1` through `1.21.11`
- one config file stored in `config/autotune_fps.json`
- one main command root: `/optimizer`
- four manual presets plus one technical off state:
  - `ultimate_performance`
  - `performance`
  - `balanced`
  - `quality`
- `off`
- optional auto-apply on world join
- a separate `recommend` action
- hardware-tier-based preset scaling
- preset application should be understandable and stable

## Out of Scope

- custom rendering engine
- direct competition with Sodium / Embeddium internals
- dynamic FPS chasing that constantly changes settings during gameplay
- shader or modpack benchmarking
- telemetry

## UX Rules

- presets should be named for player intent, not technical jargon
- applying a preset should be one action
- settings should apply explicit values, not lowering-only behavior
- "off" should stop future AutoTune changes but not silently invent a restore system yet
- `ultimate_performance` should be the absolute lowest practical visual-quality preset for maximum FPS
- `performance` should be FPS-focused, but less extreme than `ultimate_performance`
- `balanced` should be the recommended middle point for the detected hardware tier
- `quality` should be the highest safe visual-quality preset for the detected hardware tier
- tier differences should matter most for `balanced` and `quality`

## Command Surface

- `/optimizer`
- `/optimizer help`
- `/optimizer status`
- `/optimizer menu`
- `/optimizer recommend`
- `/optimizer preset ultimate_performance / performance / balanced / quality / off`
- `/optimizer preset <ultimate_performance|performance|balanced|quality|off>`
- `/optimizer autoapply <on|off>`

## Technical Rules

- keep the mod client-only
- keep the implementation compatible with standard Fabric Loom layout
- detect `LOW`, `MID`, and `HIGH` hardware tiers via CPU + RAM + GPU scoring
- store the selected preset in config so future UI can read the same state
- do not modify Max FPS / Framerate Limit, VSync, FOV, GUI Scale, Brightness, or Resolution / fullscreen mode

## Compatibility Goal

AutoTune FPS should work safely alongside common client optimization mods such as Sodium and similar mods.

- do not assume AutoTune FPS is the only optimization mod installed
- prefer compatibility over forcing settings that may conflict with other mods
- avoid changing options that are likely to be owned or overridden by other optimization mods
- detect common compatibility situations where possible and fail safely
- if a setting cannot be applied safely because of another mod, use a fallback instead of crashing
- keep this as a roadmap/spec direction unless a small safe compatibility fix is clearly needed now

## Tier Detection

CPU score:

- `1 to 4` available threads -> `0`
- `6 to 8` available threads -> `1`
- `10+` available threads -> `2`

RAM score:

- `8 GB` or less total system RAM -> `0`
- `12 to 16 GB` -> `1`
- `24 GB or more` -> `2`

GPU score:

- software/basic/integrated-looking renderer -> `0`
- unknown/older/mid renderer -> `1`
- clearly dedicated gaming/workstation renderer -> `2`

Tier result:

- total `0 to 2` -> `LOW`
- total `3 to 4` -> `MID`
- total `5 to 6` -> `HIGH`

## Fabulous Rule

Allow `Fabulous` only if all are true:

- hardware tier is `HIGH`
- no shaders are active
- renderer/vendor does not look like a software/basic compatibility renderer
- no obvious risk flags are detected

Otherwise, `HIGH` tier `quality` must fall back to `Fancy`.

## Preset Table

`LOW`

- `ultimate_performance`: Fast graphics, Off clouds, Minimal particles, 6 render distance, 5 simulation distance, 50% entity distance scaling, 0 biome blend, entity shadows Off, chunk updates Nearby
- `performance`: Fast graphics, Off clouds, Minimal particles, 8 render distance, 5 simulation distance, 50% entity distance scaling, 0 biome blend, entity shadows Off, chunk updates Nearby
- `balanced`: Fast graphics, Fast clouds, Decreased particles, 10 render distance, 6 simulation distance, 75% entity distance scaling, 2 biome blend, entity shadows Off, chunk updates Player Affected
- `quality`: Fancy graphics, Fast clouds, Decreased particles, 12 render distance, 7 simulation distance, 100% entity distance scaling, 3 biome blend, entity shadows On, chunk updates Player Affected

`MID`

- `ultimate_performance`: Fast graphics, Off clouds, Minimal particles, 6 render distance, 5 simulation distance, 50% entity distance scaling, 0 biome blend, entity shadows Off, chunk updates Nearby
- `performance`: Fast graphics, Off clouds, Minimal particles, 10 render distance, 6 simulation distance, 75% entity distance scaling, 2 biome blend, entity shadows Off, chunk updates Nearby
- `balanced`: Fancy graphics, Fast clouds, Decreased particles, 12 render distance, 8 simulation distance, 100% entity distance scaling, 3 biome blend, entity shadows On, chunk updates Player Affected
- `quality`: Fancy graphics, Fancy clouds, All particles, 16 render distance, 10 simulation distance, 125% entity distance scaling, 5 biome blend, entity shadows On, chunk updates Player Affected

`HIGH`

- `ultimate_performance`: Fast graphics, Off clouds, Minimal particles, 6 render distance, 5 simulation distance, 50% entity distance scaling, 0 biome blend, entity shadows Off, chunk updates Nearby
- `performance`: Fast graphics, Off clouds, Decreased particles, 12 render distance, 8 simulation distance, 100% entity distance scaling, 3 biome blend, entity shadows On, chunk updates Nearby
- `balanced`: Fancy graphics, Fancy clouds, All particles, 18 render distance, 10 simulation distance, 125% entity distance scaling, 5 biome blend, entity shadows On, chunk updates Player Affected
- `quality`: Fabulous if safe, otherwise Fancy; Fancy clouds; All particles; 24 render distance; 12 simulation distance; 150% entity distance scaling; 7 biome blend; entity shadows On; chunk updates Player Affected
